argument

Properties:
Name Type Attributes Default Description
name string The argument name. This options has two purposes: to document the command help, and to identify the option in the "argv.args" variable passed back to the command function.
desc string <optional>
 A description about what the argument is and what information the user is expected to supply. It is used to show the command help to the user.
type string The argument data type, either "string" or "number".
required boolean <optional>
 false Whether or not the argument is required for the command to work. If the user fails to supply every required argument, Clapp will warn them about the problem and redirect them to the command help.
default string | number <optional>
 A default value that will be passed into the "argv.args" if the user does not supply a value. This only works if "required" is set to false; it has no effect otherwise.
An argument is an option passed to the command. In the CLI sentence (See App.isCliSentence) "/testapp foo bar", "bar" would be the value of the first argument.

var arg = {
	name: 'file',
	desc: 'The file where the data will be saved',
	type: 'string',
	required: false,
	default: 'defaultfile.dat'

flag

Properties:
Name Type Attributes Description
name string The flag name. This options has two purposes: to document the command help, and to identify the option in the "argv.flags" variable passed back to the command function.
desc string <optional>
 A description about what the flag is and what information the user is expected to supply. It is used to show the command help to the user.
type string The flag data type, either "string", "number" or "boolean".
default string | number | boolean <optional>
 A default value that will be passed into the "argv.flags" if the user does not supply a value.
alias string <optional>
 A string of only one character containing the alias of the flag. If the alias of "limit" is "l", then "--limit=15" will have the same effect as "-l 15".
A flag is an option passed to the command. Unlike arguments, flags are optional by nature, meaning that you can't require the user to pass a flag. Flags always have a full name, and may have aliases (i.e: "--debug" and "-d"). In the CLI sentence (See App.isCliSentence) "/testapp foo --bar", "bar" would be the flag the user passed, and its value would be "true".
Type:
  • Object
Example
var flag = {
	name: 'limit',
	desc: 'The number of items that will be shown. Can\'t be higher than 50',
	type: 'number',
	alias: 'l',
	default: 10
}

onReply()

Source:
  • index.js, line 68
The onReply function gets called every time Clapp needs to show output to the user. It receives two parameters: msg and context. The msg parameter is the message that you need to show to the user. It can be the generated command help, if the user requests it, an error message if the user doesn't pass a valid cli sentence, or a message returned by the command function. The context is what you defined, for more information see Working-with-contexts
Example
function(msg, context) {
	createMessage(msg, context.user);
}
