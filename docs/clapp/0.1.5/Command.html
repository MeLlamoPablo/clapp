Command

new Command(name, fn, descopt, argumentsopt, flagsopt, asyncopt)

Source:
  • Command.js, line 7
A Command that can be bound to an App. A command represents a single function that achieves a single purpose. The command can receive arguments and flags, and can return a string that will be redirected to the app output (See: onReply).
Example
var foo = new Clapp.Command(
	'foo',
	function(argv, context) {
		console.log('foo was executed');
	},
	'does foo'
	[
		{
			name: 'testarg',
			desc: 'A test argument',
			type: 'string',
			required: false,
			default: 'testarg isn\'t defined'
		}
	],
	[
		{
			name: 'testflag',
			desc: 'A test flag',
			alias: 't',
			type: 'boolean',
			default: false
		}
	]
);

app.addCommand(foo);
Parameters:
Name Type Attributes Default Description
name string The command's name. The command will be invoked by writing the app prefix followed by a space and the command's name.
fn function The function that will be executed when the command is invoked. The function receives the following params: argv and context. For more information, see Defining-the-command-function.
desc string <optional>
 A description of what the command does.
arguments Array.<argument> <optional>
 An array with every argument supported by the command. See argument.
flags Array.<flag> <optional>
 An array with every flag supported by the command. See flag.
async boolean <optional>
 false Whether or not the command's function is asynchronous. If it is, it will receive a third parameter: callback, that you will need to call when the process is over. See Defining-the-command-function.
