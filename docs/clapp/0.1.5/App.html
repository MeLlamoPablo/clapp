App

App

new App(options, onReply, commandsopt)

Source:
  • index.js, line 9
A command line app. An App can parse an input with parseInput, and if it's correct (i.e: refers to an existing command and provides the required options), the command will be executed. An App needs an onReply function to be able to communicate with the user.
Example
const Clapp = require('clapp');
var myApp = new Clapp.App(
	{
		name: 'Test App',
		desc: 'An app created with Clapp!',
		prefix: '/testapp', // Commands will have this structure: /testapp foo --bar
		version: '1.2.0'
	},
	function(msg, context) {
		// Called when the App shows an output
		console.log(msg);
	},
	[myCommand1, myCommand2]
);

myApp.addCommand(myCommand3);
Parameters:
Name Type Attributes Description
options object
Properties
Name Type Attributes Description
name string The app's name. This isn't necessarily the same as the prefix. Its only purpose is to be shown to the user in case of syntax error, or when they request the app help.
desc string A description of what the app does. This will be used to document the app's help.
prefix string The app's prefix. It will be used by parseInput to determine the user's intention to run a command corresponding to this App instance.
version string <optional>
 The app's version. It will be used to document the app's help.
onReply onReply A function that allows the App to communicate with the end user. When the App needs to show an output, the onReply function will be executed.
commands Array.<Command> <optional>
 An array of commands that will immediately be bound to the app. Additional commands can be later bound using addCommand.

isCliSentence(sentence)

Source:
  • index.js, line 283
Validates an input to find out whether or not it is a CLI sentence. A CLI sentence (valid or not) is a string that begins with the app prefix. A valid CLI sentence is a CLI sentence that does not result in an error upon parsing.
Example
app.isCliSentence('/testapp foo --bar'); // True
app.isCliSentence('Hello, world!')       // False
Parameters:
Name Type Description
sentence string The string to test.

parseInput(input, context)

Source:
  • index.js, line 136
Parses an input CLI sentence (See isCliSentence) and performs actions accordingly: if the sentence is a valid command, that command is executed. If it is an invalid CLI sentence, the user is warned about the problem. If the user passes the "--help" flag, they are prompted with the app general help, or the command specific help. Please note that the input is not sanitized. It is your responsibility to do so. It is also reccommended that you make sure that the input is a CLI sentence (valid or not) by using isCliSentence. Otherwise, Clapp will log an error.
Example
app.parseInput('/testapp foo');        // Executes `foo`
app.parseInput('/testapp foo --bar');  // Executes `foo` passing the --bar flag
app.parseInput('/testapp foo --help'); // Shows the command help for `foo`
app.parseInput('/testapp --help');     // Shows the app help
app.parseInput('Not a CLI sentence');  // Does nothing, logs a warning.
Parameters:
Name Type Description
input string A CLI sentence. See isCliSentence.
context * The context to retrieve later. See Working-with-contexts.
