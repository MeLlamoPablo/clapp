Flag

Flag

A flag is an option passed to a Command. Unlike arguments, flags are optional by nature, meaning that you can't require the user to pass a flag. Flags always have a full name, and may have aliases (i.e: --debug and -d). In the CLI sentence (See App.isCliSentence) /testapp foo --bar, bar would be the flag the user passed, and its value would be true.

Example
let flag = new Clapp.Flag({
	name: "limit",
	desc: "The number of items that will be shown. Can\'t be higher than 50",
	type: "number",
	alias: "l",
	default: 10
});

Properties
Name Type Attributes Default Description
name string

The Flag name. This options has two purposes: to document the command help, and to identify the option in the argv.args variable passed back to the command function.
desc string

A description about what the Flag is and what information the user is expected to supply. It is used to show the command help to the user.
type string

The Flag data type, either string or number, or boolean
default string | number | boolean

A default value that will be passed into the argv.args if the user does not supply a value.
alias string <optional>

A string of only one character containing the alias of the flag. If the alias of limit is l, then --limit=15 will have the same effect as -l 15.
caseSensitive boolean <optional>
 true

Whether or not the flag is case sensitive. This doesn't affect aliases, which are always case sensitive.
validations Array.<validation> <optional>

An array with every validation check that you want to perform on the user provided value. See validation.
